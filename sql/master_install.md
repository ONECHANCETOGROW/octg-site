# Database Installation Guide

This document describes the structure of the ONE master SQL file (`master_install.sql`), which is generated by concatenating migrations 001 through 010.

## Installation Order

1. `001_contact_messages.sql` (contact.php submissions)
2. `002_demo_requests.sql` (book-demo.php submissions)
3. `003_newsletter_subscribers.sql` (newsletter signups)
4. `004_audit_requests.sql` (audit.php submissions)
5. `005_admin_users.sql` (Admin users table)
6. `006_leads.sql` (Unified CRM leads, depends on admin_users)
7. `007_email_notifications.sql` (Email settings and logs)
8. `008_activity_log.sql` (Admin activity logging)
9. `009_website_optimizer.sql` (AI website optimizer tools)
10. `010_knowledge_engine.sql` (Knowledge Graph data structure overrides)

## Tables and Relationships

### 1. Unified CRM (`leads` & `lead_activity`)
- **leads**: The central CRM table containing normalized lead data from Contact, Demo, and Audit forms.
  - *Foreign Key*: `assigned_to` -> `admin_users(id)`
- **lead_activity**: Tracks notes and status changes for leads.
  - *Foreign Key*: `lead_id` -> `leads(id)`

### 2. Admin & Auth (`admin_users` & `activity_log`)
- **admin_users**: Authentication for the private backend. Contains roles (super_admin, admin, editor, etc.).
- **activity_log**: Tracks global actions taken by admin users.

### 3. Website Optimizer (`audit_reports`, `audit_page_scores`, `audit_issues`, `draft_improvements`, `optimization_history`, `optimizer_settings`)
- **audit_reports**: Represents a single crawl/run.
- **audit_page_scores**: Scores assigned to individual pages during a report.
  - *Foreign Key*: `report_id` -> `audit_reports(id)`
- **audit_issues**: Specific problems identified during the crawl.
  - *Foreign Key*: `report_id` -> `audit_reports(id)`
- **draft_improvements**: Suggested fixes for an issue.
  - *Foreign Key*: `issue_id` -> `audit_issues(id)`
- **optimization_history**: Historical record of applied fixes.
  - *Foreign Key*: `draft_id` -> `draft_improvements(id)`

### 4. Direct Form Tables (Archival)
- **contact_messages**: Raw submissions from the contact form.
- **demo_requests**: Raw submissions from the demo wizard.
- **audit_requests**: Raw submissions from the free audit form.
- **newsletter_subscribers**: Simple email collection for the newsletter.

### 5. Email & Notification System (`email_settings`, `email_logs`)
- **email_settings**: Singleton table storing the site's outbound email config (SMTP credentials).
- **email_logs**: Log of every email dispatched by `notification-service.php`.

## Indexes
Key indexes are configured on timestamps (e.g., `created_at`), `status` fields, and all foreign keys (e.g., `lead_id`, `report_id`, `assigned_to`) to ensure query speed as the dataset grows.

## Instructions
To perform a fresh install, import `master_install.sql` into an empty MySQL/MariaDB database. Then, navigate to `/admin/setup.php` in your browser to create the first `super_admin` user.
